# Saozi UI/UX & Service Architecture (V2)

## 1. 3대 역할별 완벽한 분리 (Role-Based Architecture)
하나의 앱에서 접속자의 권한(역할)에 따라 진입점과 제공되는 기능이 완전히 달라야 합니다. 기존처럼 모두가 동일한 `/jobs` 피드를 보는 구조를 탈피합니다.

### A. 파티 주최자 (Organizer) Flow
**목표:** 필요한 모델을 빠르고 직관적으로 뽑는 것.
- **홈 화면 (`/organizer`):** 
  - 내가 올린 공고 리스트와 현재 지원 현황(예: 3명 지원 대기중) 대시보드.
  - 하단 고정 FAB(Floating Action Button): `+ 새 공고 작성`
- **지원자 심사 뷰 (`/organizer/jobs/[id]/applicants`):**
  - 카드 스와이프(Tinder style) 또는 큼직한 세로 스크롤 카드.
  - **핵심 정보:** 실시간 카메라 인증샷, 키/몸무게, 나이.
  - 액션: `[승인]` / `[거절]` 버튼 명확화.

### B. 모델 (Model) Flow
**목표:** 좋은 알바를 찾고, 현장 인증으로 빠르게 지원하는 것.
- **홈 화면 (`/model`):**
  - 현재 모집 중인 공고 피드. (금액이 가장 크게 보이는 UI)
- **내 프로필 (`/model/profile`):**
  - 기본 카메라 프로필 사진, 키, 몸무게, 나이, SNS 링크 관리.
- **지원 플로우 (1 Thing 1 Page):**
  - 공고 상세 -> [지원하기] -> [내 프로필 확인] -> [실시간 카메라 촬영] -> [지원 완료].

### C. 에이전시 / MD (Agency) Flow
**목표:** 소속 모델들을 관리하고, 여러 공고에 대리 지원하여 매칭 수수료를 극대화하는 것.
- **홈 화면 (`/md`):**
  - 소속 모델 리스트(Model Roster) 및 대리 지원 현황 대시보드.
- **대리 지원 플로우 (해결 과제 포함):**
  - MD가 공고를 보고 -> 소속 모델 A를 선택하여 지원.
  - *해결책:* MD가 대리 지원 시, 모델의 카카오톡/문자로 **"실시간 카메라 인증 링크"**가 발송되게 하거나, MD 권한에 한해 **최근 보증된 프로필 사진**으로 대체하여 지원 가능하도록 로직 분리.

## 2. 모바일 앱 네비게이션 구조 (Toss Style App Bar)
웹이지만 네이티브 앱처럼 동작하도록 하단 네비게이션 탭(Bottom Tab)을 도입합니다.
- **주최자 탭:** [홈(내 공고)] / [알림] / [내 정보]
- **모델 탭:** [알바 찾기] / [지원 내역] / [내 프로필]
- **MD 탭:** [알바 찾기] / [소속 모델 관리] / [지원 내역] / [내 정보]

## 3. UI/UX 디자인 토큰 강화
- **Surface & Elevation:** 카드는 배경색(`#f2f4f6`) 위에 하얀색(`#ffffff`)으로 둥글게(`rounded-[24px]`) 띄우고, 미세한 투명 그림자(`shadow-sm`)를 줍니다.
- **Typography:** 가장 중요한 숫자(금액)는 `32px` 이상의 크기와 `font-bold`로 압도적으로 보여줍니다. 부가 설명은 `14px` `text-gray-500`으로 뺍니다.
- **Micro-interactions:** 페이지 전환 간 끊김 없는 라우팅 애니메이션, 버튼 탭 시 햅틱 효과 모방(`active:scale-[0.98]`).
