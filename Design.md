# Saozi UI/UX Design System (Toss Style)

## 1. 핵심 디자인 철학
토스(Toss) UI의 핵심 원칙인 **"1 Thing / 1 Page" (한 화면에 하나의 행동만 유도)**와 인지 부하를 최소화하는 **"쉬운 언어와 카드형 레이아웃"**을 적극 도입하여 앱을 전면 개편합니다.

## 2. 시각적 가이드라인 (Visual Guidelines)
- **컬러 팔레트:** 
  - Primary: `#3182f6` (Toss Blue)
  - Background: `#f2f4f6` (연한 쿨그레이)
  - Card/Surface: `#ffffff`
  - Text Primary: `#191f28` (완전한 검은색 지양)
  - Text Secondary: `#8b95a1`
- **타이포그래피:** 크고 굵은 제목(Display), 명확한 계층 구조. 금액과 숫자는 눈에 띄게 크게 배치.
- **모양 및 간격 (Shape & Spacing):** 
  - 모든 카드와 모달의 모서리는 둥글게 (`border-radius: 16px ~ 24px`).
  - 답답하지 않게 컨텐츠 밖의 여백(Padding)을 넓게(`20px` 이상) 사용.
- **버튼 (CTA):** 화면 하단에 고정된 넓고 눈에 띄는 파란색 버튼(Sticky Bottom Button).

## 3. 화면별 개편 사항 (Flow Redesign)

### A. 메인 랜딩 (로그인 / 역할 선택)
- 기존: 작은 텍스트와 좁은 폼.
- **개편:** 최상단에 큰 인사말("어떤 역할로 시작할까요?"). 각 역할을 큼직한 카드로 분리하여 터치 영역 확대. 복잡한 폼은 역할을 선택한 다음 단계에서 노출.

### B. 공고 목록 (Feed)
- 기존: 단순한 리스트.
- **개편:** '토스 송금 내역'처럼 금액(일당)을 가장 크고 굵게 표시. 지역과 시간은 작고 깔끔한 태그나 뱃지로 정돈.

### C. 공고 상세 및 지원 흐름 (1 Thing 1 Page 적용)
- 기존: 한 화면에 공고 내용 + 버튼 존재.
- **개편:** 
  1. 공고 상세 페이지: 깔끔한 요약과 하단 고정 [지원하기] 버튼.
  2. 프로필 확인 모달/페이지: "이 프로필로 지원할까요?" (지원할 프로필 정보 큼직하게 노출).
  3. 실시간 카메라 페이지: 프로필 확인 후 오직 '사진 촬영'에만 집중하도록 분리.

### D. 인터랙션 및 애니메이션 (Micro-Interactions)
- 페이지 전환 시 부드러운 페이드인/슬라이드인.
- 버튼 터치 시 즉각적인 눌림 효과(`scale-95`, `opacity-90`).
- 실시간 촬영 후 로딩 상태 명확히 제공.

## 4. 컴포넌트 아키텍처
- `StickyBottomButton`: 토스 앱의 시그니처인 하단 고정 플로팅 버튼. SafeArea 대응.
- `TossCard`: 여백이 넉넉하고 둥근 백그라운드 카드.
- `AmountText`: 일당/금액 전용 타이포그래피.
